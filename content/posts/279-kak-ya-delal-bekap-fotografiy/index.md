Исторически мы с женой используем [Flickr](https://flickr.com/) для хранения фотографий и накопили значительный семейный альбом, который очень не хочется потерять по воле случая.

Поэтому задумались, куда и как лучше зеркалировать фотографии для пущей сохранности. В рабочих проектах мы используем [облачный S3](https://yandex.cloud/ru/docs/glossary/s3) для хранения статических файлов. Один хороший SRE подсказал, что этот способ подойдет и для хранения фотографий — дешево и сердито.

С хранилищем определились. Теперь нужно понять, как перегнать данные из Flickr в S3. К слову, про "перегонку данных". А вы знали, что Amazon в течение нескольких лет [предоставлял](https://www.cnbc.com/2024/04/17/aws-stops-selling-snowmobile-truck-for-cloud-migrations.html) специальный сервис по миграции данных в датацентры Amazon с помощью... грузовиков? Но в итоге сервис не [взлетел](https://www.cnbc.com/2024/04/17/aws-stops-selling-snowmobile-truck-for-cloud-migrations.html), вернее, не поехал :)

Мне не нужно перемещать петабайты данных, поэтому небольшой скриптик вполне подойдет. Высокоуровнево мне нужно сделать три шага:
1. Забрать данные через [Flickr SDK](https://www.npmjs.com/package/flickr-sdk).
2. Залить полученные данные в S3 Яндекс Облака через [AWS SDK](https://www.npmjs.com/package/@aws-sdk/client-s3).
3. Написать [небольшой скрипт](https://github.com/tarmolov/flickr-backup) с необходимой логикой.

После небольшой отладки скрипт заработал на моем ноутбуке. Но это только полдела. Необходимо, чтобы скрипт работал по расписанию без лишних затрат моей [ментальной энергии](https://tarmolov.ru/posts/32-ekonomiya-mentalnoy-energii-cherez-avtomatizatsiyu/).

Вот тут и пригодился мой [NAS-сервер](https://ru.wikipedia.org/wiki/NAS) Synology. Обычно я его использую как хранилище для фильмов, но сервер способен решить и более интересные задачки:
1. С помощью [docker-контейнера](https://tarmolov.ru/posts/51-tekuschaya-sistema-deploya-docker-obrazami/) настроил окружение для своего скрипта.
2. Далее через встроенный механизм Task Scheduler настроил ежедневный запуск своего скрипта вот такой командой:
```
docker exec flickr-backup /bin/bash -c "SECRETS_PATH=/etc/flickr-backup/secrets.json node /usr/local/flickr-backup/index.js"
```

Не зря Рей Далио [советует всем учить программирование](https://tarmolov.ru/posts/274-rey-dalio-printsipy/). Программирование расширяет ваши возможности и позволяет эффективно решить прикладные задачи не только для рабочих, но и для личных нужд. Например, забекапить свои личные фотографии ;)
